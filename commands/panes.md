# /panes -- list active Cursor panes

## What it does

Reads the CPC-OS pane registry at `~/.claude/state/cpc_os_registry.json`
and prints a tabular view of every known pane:

```
PANE-ID                STATUS    TASK                              CWD                                LAST-HB
test-pane-1            active    testing CPC-OS                    C:/Users/User/.claude/skills/...   12s ago
pane-2                 stale     dataset-full-ingestion-and-build  C:/Users/User/.claude/skills/...   ~6m ago
pane-3                 dead      old benchmark sweep               C:/Users/User/.claude/skills/...   --
```

Status semantics:
- `active`  -- heartbeat within the last 300 s
- `stale`   -- heartbeat older than 300 s (demoted by `mark_stale_panes`)
- `dead`    -- explicitly marked dead via `registry.mark_dead(pane_id)`

## Usage

```
/panes                          # list all
/panes --active-only            # filter to active panes
/panes --json                   # machine-readable
```

## Programmatic equivalent

```python
from modules.cpc_os import PaneRegistry, mark_stale_panes

reg = PaneRegistry.load()
mark_stale_panes(reg)                # demote anything past threshold
for rec in reg.panes.values():
    print(rec.pane_id, rec.status, rec.task, rec.last_heartbeat_at)
```

## Crash recovery (session snapshot)

Every SessionStart writes a human + machine readable manifest to
`~/.claude/state/session_snapshot.md` (generated by
`modules/cpc_os/snapshot.py` right after the pane registers). It lists
every live pane with its repo, cwd, task, current git commit, a resume
command, and the time it opened.

After a crash -- Cursor died, the host rebooted, every pane gone --
restore from that one file:

```
1. Make sure claude.exe is fully dead.
2. Open ~/.claude/state/session_snapshot.md
3. For each PANE, run its `Resume:` line in its own terminal:
   - claude --resume <id>   -> exact session restored (high confidence;
     shown when the pane's session_id was captured at open).
   - cd "<cwd>" && claude    -> new session in the right repo (safe
     fallback when no session_id is known -- never a guess).
```

The snapshot is regenerated on every SessionStart, so it always reflects
the panes that were open at the last open event. Dead panes are omitted;
stale panes (no heartbeat for >300 s) are listed with a `[stale]` flag so
you can decide whether to restore them. A trailing fenced ```json block
carries the same data machine-readably for tooling.

Programmatic:

```python
from modules.cpc_os import generate_snapshot

path = generate_snapshot()   # writes the .md, returns the path
```

## Pairs with

- `/switch-session` -- route a restart/switch intent to a specific pane,
  using the same registry as source of truth. Both commands refuse to
  act on a pane that the registry doesn't recognize.
