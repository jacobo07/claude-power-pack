# Lateral-Thinking Skill -- Empirical Scoring Protocol

**Sealed**: 2026-05-26. **Pass-gate authority**: skill-completion-standard.md C1 + C2.

This document defines the Owner-blind-eval scoring procedure for the
`lateral-thinking` skill. Without a measured aggregate, C1 (declared
pass-gate) and C2 (side-by-side comparison) are unsatisfied, and the
skill cannot claim S+++.

---

## Pass-gate (declared pre-build, not retrofitted)

**Metric**: `aggregate(novelty + depth + actionability)_with >=
aggregate(without) + 1.5`

- N=5 prompts spanning 5 problem domains (algorithmic, product-ux,
  system-design, debugging, creative).
- Each prompt is presented in two arms: **with** the LT skill card in
  `additionalContext` vs **without** (only any other independent JIT
  advisory like vague-lint or arch-check).
- The agent's response in each arm is scored by the Owner on 3 axes,
  1-5 each, max 15 points per (prompt x arm) pair.
- Aggregate = sum across all 5 prompts. Max aggregate = 75 per arm.

**Threshold delta**: 1.5 points. Empirically chosen as 10% of the
15-point scale -- below this delta the skill is statistically
indistinguishable from no-skill on this fixture and KILL-cycles
(per Reality Contract) without false promotion.

---

## Three scoring axes (1-5 each)

### 1. Novelty
*Does the response surface a candidate direction the obvious framing
would NOT have produced?*

| Score | Interpretation |
|---|---|
| 1 | Pure restatement of the prompt + the most obvious technique. |
| 2 | One predictable next-step (already considered by the prompter). |
| 3 | One non-obvious angle. |
| 4 | Multiple non-obvious angles, at least one unfamiliar. |
| 5 | At least one angle the prompter would not have reached themselves. |

### 2. Depth
*Does the response produce concrete artifacts (lists, mappings,
analogies, base axioms) or just labels?*

| Score | Interpretation |
|---|---|
| 1 | Labels only ("apply inversion", "first principles"). No artifact. |
| 2 | One frame applied with one example. |
| 3 | Two frames applied, each with one concrete artifact. |
| 4 | Three frames applied with structured artifacts (lists, tables, analogies). |
| 5 | Three frames + audit-trail block + scoring rationale visible. |

### 3. Actionability
*Can the prompter act on the response within one work session?*

| Score | Interpretation |
|---|---|
| 1 | Pure philosophy, no executable step. |
| 2 | Vague suggestion, requires re-prompting to convert to action. |
| 3 | One actionable candidate direction. |
| 4 | Two-plus actionable candidates with rough effort estimates. |
| 5 | Two-plus actionable candidates + ordered test sequence + observability. |

## How to score

1. Open the latest `vault/ceps/lt_empirical_canonical_<ts>.json`
   fixture (generated by `tools/lt_empirical_regen.py`).
2. For each prompt, generate the agent's response under both arms
   (with / without LT card). The fixture carries the
   `additionalContext` that the agent would have seen; replay each
   pair through any Claude Code pane and capture the response.
3. Score the responses blind -- ideally without knowing which arm is
   which. The fixture is symmetrically structured to permit this:
   relabel `with` / `without` to `A` / `B` before reading the
   responses if needed.
4. Fill `owner_score_with` / `owner_score_without` for each prompt
   in the fixture JSON. Each axis takes an integer 1-5.
5. Compute aggregate per arm: sum of (novelty + depth +
   actionability) across all 5 prompts.
6. Pass-gate check: `aggregate_with >= aggregate_without + 1.5`.
   - PASS: the skill ships at S+++. Update SCS C1+C2 evidence rows.
   - FAIL (delta < 1.5): iterate the LT skill body / regex /
     scoring rubric for up to 3 cycles per the iteration protocol.
     Cycle 3 FAIL -> skill is KILL-candidate per Reality Contract.

---

## Why blind by Owner, not auto-scored

The agent that built the skill cannot also score it without bias --
self-scoring would defeat the empirical pass-gate. The Owner is the
single human-in-the-loop role this gate requires. Auto-scoring by a
separate LLM is forbidden because:

1. It introduces a confound (which LLM, which prompt, which
   temperature).
2. It would make the pass-gate non-deterministic.
3. The Owner is the consumer of the skill's outputs in production;
   their score is the ground truth.
