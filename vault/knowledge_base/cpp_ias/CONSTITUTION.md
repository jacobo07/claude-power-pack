# CPP-IAS — CONSTITUTION

> Binding rules for every dataset in this family. A dataset that violates any article is not
> sealed. Read before authoring any Part.

## Article I — The Duplicate-to-Advantage law (supreme)
A duplication is never merely deleted. Every overlap with an existing PP system is converted into
a **second-order capability** that makes that system (and ≥1 other) more valuable. A dataset whose
only novelty is a new name for an existing object is REJECTED (`naming-only novelty` = REJECT).

## Article II — The ensemble-object boundary
Every IAS dataset governs the **ensemble** of PP systems, never a single resource type. The instant
a dataset would govern models (CO owns), panes (PM owns), knowledge nodes (GK owns), one decision
(DRK owns), one duplication (D2A owns), or one session's cost (CO owns), it is duplicating a parent.
The distinct institutional object must be provable in `NON_DUPLICATION_LEDGER.md`.

## Article III — Strengthen, never replace
Every dataset strengthens ≥2 existing systems (priority datasets 5–20), recorded in
`SYSTEM_STRENGTHENING_MATRIX.md` with a concrete `evidence required` artifact. A strengthening claim
without checkable evidence is marked HYPOTHESIZED and never stated as fact.

## Article IV — Compounding or orphan
Every dataset declares what it receives, produces, who consumes it, and which system it amplifies
(`COMPOUNDING_GRAPH.md`). A dataset with no inbound and no outbound edge is an orphan → REJECTED.

## Article V — Epistemic honesty
Every load-bearing claim is marked: `DEFINED · OBSERVED · DERIVED · HYPOTHESIZED · NORMATIVE ·
EXPERIMENTAL · CONTESTED · UNRESOLVED`. A mechanism asserted without a level defaults to HYPOTHESIZED.
No dataset claims a system "works" or is "done" — only that evidence of a given level licenses a claim.

## Article VI — Depth without padding
Depth target: **1,200–1,500 real words per Part**, floor never lowered to fit a draft; **15–35 Parts
by real complexity** (never a quota). A Part is REJECTED if it is: repetitive · generic · padded to a
count · a summary instead of a mechanism · missing failure modes / evaluation / interfaces · or
re-narrating a parent system instead of cross-referencing it.

## Article VII — Domain purity
Exclusively for Claude Power Pack. Zero contamination from CommonWealth Ops, InfinityOps, ecommerce,
ads, stores, operators, supply chains, or any external ecosystem. The CUAE reference is a **depth**
benchmark only — never a source of entities, metrics, processes, or terminology. A Domain Contamination
Audit (0 hits) runs before sealing each dataset and before sealing the compendium.

## Article VIII — Reality contract
No production code this mission (`PR-CPP-ACI-SCIENCE-BEFORE-IMPLEMENTATION-001`): datasets are the
substrate; implementation is a later mission. Zero placeholders, stubs, TODOs, or nominal Parts.
Every dataset names its Dataset-to-Runtime translation candidates (routers/gates/hooks/registries/
schemas/evaluators/metrics/agents) as *specification*, not code.

## Article IX — Governed self-evolution
No IAS system may recommend self-modification of the estate without an evidence threshold, validation,
versioning, and rollback (`T-SQI-SELF-EVOLUTION-UNCONTROLLED-001`). IAS proposes; the Owner promotes.

## Article X — No estate mutation
This mission does not alter the Claude Power Pack repo. Corpus lands in this output dir only. Never
`git add -A` in PP. The estate is read as substrate, strengthened by specification, never edited here.

## Article XI — Only SEALED counts
A dataset is done only when every Part passes its gates AND a Domain Contamination Audit AND its
strengthening evidence exists. `CONTENT_COMPLETE` ≠ `SEALED`. Completeness is judged on inspected
content, not file existence.

## Dataset state machine
`DISCOVERED → MAPPED → CONTRACTED → OUTLINED → IN_PROGRESS → CONTENT_COMPLETE → INTEGRATED →
EVALUATED → HARDENED → SEALED`.

## Epistemology vocabulary (for Article V)
- **DEFINED** — true by construction/definition within this corpus.
- **OBSERVED** — measured against the real estate (a file, a count, a module).
- **DERIVED** — follows by stated reasoning from DEFINED/OBSERVED premises.
- **HYPOTHESIZED** — plausible mechanism, not yet evidenced.
- **NORMATIVE** — a rule/should, not a fact.
- **EXPERIMENTAL** — proposed to be tested; carries its test.
- **CONTESTED** — evidence points both ways; both sides stated.
- **UNRESOLVED** — open question, named as such.
