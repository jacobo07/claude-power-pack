# CPP-IAS — NON-DUPLICATION LEDGER

> The Duplicate-to-Advantage verdict for every candidate family. No IAS dataset is sealed
> until this ledger proves its institutional object is distinct from every parent.
> Verdict vocabulary (D2A): `BUILD · MERGE-INTO-FAMILY · SUBENGINE · EXTEND · COMPOSITION ·
> REFERENCE · REJECT`. Disposition of the 150 candidates → the 14 canonical datasets.

## Format
`Candidate → closest parent(s) → overlap → UNIQUE ensemble object → verdict → routed to`

## The ledger law
14 datasets survive; ~90 candidates fold into them as subengines/merges; ~46 route to REFERENCE.
Every surviving dataset carries, in its own text, the specific candidates it absorbs and the
parents it strengthens. **A dataset that cannot name a distinct ensemble-level object is REJECTED.**
