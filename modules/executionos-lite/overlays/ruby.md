# Overlay: Ruby (MC-OVO-30-LANG — 5 of 8 language overlays)

Loads when CWD contains `Gemfile`, `Gemfile.lock`, `*.rb`, `config.ru`, `Rakefile`, or a `lib/<gem>.rb` entry point. ~240 tokens.

- **Ruby version pinning:** `.ruby-version` at repo root declaring `3.3.x` or newer (never unpinned). Managed via `rbenv`, `asdf`, or `chruby`; `rvm` discouraged for new projects. `ruby --version` in CI logs to catch drift.
- **Bundler discipline:** `Gemfile` declares every top-level dep with pinned major (`gem "rails", "~> 7.2"`). `Gemfile.lock` committed for apps, NOT for gem libraries (libraries let consumers resolve). `bundle config set --local deployment true` in production; `bundle exec` prefix on every script invocation to use lockfile versions, never system gems.
- **Framework routing:** Rails for monolith web apps; Sinatra or Roda for micro-services; Sidekiq for background jobs (NOT `rescue-rb`, maintenance is worse). For CLI → Thor (preferred) or OptionParser (stdlib). For gem packaging → `bundle gem <name>` scaffold.
- **Naming & conventions:** `snake_case` for methods/vars, `CamelCase` for classes/modules, `SCREAMING_SNAKE` for constants. Predicate methods end in `?` (`valid?`), destructive methods end in `!` (`save!`). Private methods declared via `private` keyword (not `private :method_name` — harder to scan).
- **Exceptions over nil:** raise typed exceptions (`class MyError < StandardError; end`) for RECOVERABLE-but-exceptional errors. Return `nil` only when callers routinely check for absence (e.g., `find_by`). Bare `rescue` (`rescue => e`) banned — ALWAYS specify the class: `rescue ArgumentError, TypeError => e`. Empty `rescue` blocks are a delivery-blocker.
- **Duck typing with defensive checks at boundary:** `respond_to?(:method_name)` only at API boundaries; inside the module, assume the type. Sorbet (`sorbet` gem) or RBS (Ruby 3 stdlib) for gradual static typing on critical paths — not required everywhere but MUST be on public library APIs.
- **Concurrency:** MRI Ruby has a GVL — threads don't give you CPU parallelism, only I/O overlap. `Thread` for I/O-bound concurrent work; `Process.fork` + `Parallel` gem for CPU parallelism. `async` / `async-http` for modern fiber-based cooperative concurrency. NEVER mix `Thread` and fibers in the same request path — state confusion.
- **Rails specifics (if Rails detected):** strong params enforced at controller boundary; `ActiveRecord::Base.transaction { ... }` for multi-row writes; N+1 query detection via `bullet` gem in dev/test; `ActionController::Parameters.permit` strict; CSRF on ALL non-API forms; `rails credentials:edit` for secrets, NEVER commit `master.key`.
- **Testing:** RSpec over Minitest for apps/gems where ergonomics matter. Minitest for stdlib-only projects. Factories via `factory_bot` (NOT fixtures for anything beyond seed data). `parallel_tests` for test-suite speedup. Coverage via `simplecov` + CI-enforced threshold.
- **Linters:** `rubocop` (community style) + `rubocop-rails` / `rubocop-rspec` plugins as applicable. `rubocop-performance` flags known slow-path gotchas. `.rubocop.yml` committed; disable rules ONLY with an inline comment stating why.
- **Dependencies:** `bundle outdated` weekly (or via Dependabot/Renovate). `bundle audit` for vulnerability scanning (CVE db). For apps: `bundle install --frozen` in CI (fails if Gemfile.lock drifted).
- **Verification gate:** `bundle install --frozen` + `bundle exec rubocop --fail-level error` + `bundle exec rspec` (or `rake test`) + `bundle audit check --update` must ALL exit 0. For Rails add `bundle exec rails test:system` for integration coverage.

### Fragile-language note (Ley 25 context)

Ruby is Python's closest analog in the fragility bucket — dynamic typing, GVL-bound concurrency, mature GC but no OTP-style supervision. For daemon/service code scoring ≥4 on fragility criteria (long-running + concurrent + fault-tolerant + stateful), Elixir beats Ruby even when the team knows Rails better: Sidekiq's retry semantics are weaker than Oban/Broadway; ActiveJob's queue abstraction leaks everywhere. Ruby wins on: rapid web-app prototyping (Rails convention-over-configuration), gem ecosystem maturity, team familiarity in legacy shops. Prefer Elixir for green-field concurrent backends.
