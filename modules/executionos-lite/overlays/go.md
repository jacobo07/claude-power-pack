# Overlay: Go (MC-OVO-30 — 2 of 8 language overlays)

Loads when CWD contains `go.mod`, `go.sum`, `*.go`, or a `cmd/` dir with Go entry points. ~260 tokens.

- **Module discipline:** every new repo starts with `go mod init <path>`. Module path MUST match the intended import path — refactoring module paths later breaks every downstream consumer. Use `go.mod` `replace` directives only for local dev, never commit `replace` to main.
- **Package layout:** `cmd/<binary>/main.go` for executables (one main per binary). Library code in `pkg/` or at module root. Internal-only code in `internal/` (compiler enforces visibility). Keep `main` packages thin — parse flags, wire dependencies, call into library.
- **Concurrency:** goroutines are cheap, unbounded goroutines are not. Every goroutine needs: (1) a context for cancellation, (2) a bounded lifetime (defer-cleanup or select-on-ctx.Done()), (3) a way for the caller to know it finished (WaitGroup, errgroup, or a done channel). Goroutine leaks are the #1 Go prod incident — `runtime.NumGoroutine()` in debug endpoints catches them.
- **Context propagation:** functions that do I/O or may block MUST accept `ctx context.Context` as the first parameter. Never store a context in a struct. Pass context all the way down to the HTTP client, DB driver, and any `<-chan`/`time.After` select. `context.Background()` only at `main` / tests; `context.TODO()` is a migration marker.
- **Error handling:** return `error` as the LAST value. Wrap with `fmt.Errorf("context: %w", err)` for unwrapping. Check errors at every call site — `//nolint:errcheck` is banned except for known-safe writes (`fmt.Fprintln(os.Stderr)`). Use `errors.Is` / `errors.As` for branching on sentinel/typed errors; string matching on `err.Error()` is a bug.
- **Nil safety:** nil interface values are NOT equal to their typed-nil equivalents (`var err *MyError; return err` returns a non-nil interface wrapping a nil pointer). Return bare `nil` on success paths. Use `golangci-lint nilnil` + `nilaway` or run tests with `-race` + nil-input coverage.
- **Channels:** direction-typed chans (`chan<- T` / `<-chan T`) in function signatures document intent. Always close a chan from the SENDER, never the receiver. `select` with a `default` branch for non-blocking sends; without default for blocking wait. Unbuffered chans are synchronization points — use them when you need rendezvous, not when you need a queue.
- **Testing:** `_test.go` co-located with the code. `t.Parallel()` on independent tests. Table-driven tests are idiomatic (`tests := []struct{ name string; ... }`). `t.Cleanup()` beats `defer` in tests (runs even on t.Fatal). Coverage via `go test -cover ./...` — target ≥70% on library packages, ≥50% on binary packages.
- **Race detector:** `go test -race ./...` in CI. Any `-race` failure is delivery-blocking — mutexes are cheap, data races are not.
- **Lint:** `gofmt -d .` (MUST be a no-op), `go vet ./...`, `staticcheck ./...`, `golangci-lint run` in CI. Pre-commit hook required.
- **Dependencies:** `go mod tidy` before every commit — no stale `require`/`indirect` lines. `go get -u` only in an explicit upgrade PR (not mixed with feature work). `go mod vendor` only if the deployment target lacks network at build time.
- **Build:** `go build -trimpath -ldflags "-s -w"` for release binaries (trim paths + strip debug → reproducible, smaller). Cross-compile via `GOOS=linux GOARCH=amd64 go build`. Cgo disabled by default unless required (`CGO_ENABLED=0`) — pure-Go binaries are statically linkable and deploy-portable.
- **Verification gate:** `gofmt -l . | wc -l` returns 0 AND `go vet ./...` exit 0 AND `go test -race -cover ./...` passes AND `staticcheck ./...` exit 0. All four. Delivery-blocking on any single failure.

### Fragile-language note (Ley 25 context)

Go sits closer to "resilient" than Rust or Python because of goroutines + channels + context.Context as idioms, but it is NOT at BEAM-level for supervised fault-tolerance. For daemon/service code scoring ≥4 on fragility criteria (concurrent + long-running + multi-process + distributed), Elixir/OTP still wins on supervision trees and hot code reload. Go wins on CPU-bound work, static binaries, and "one binary per service" deployment simplicity. Pick Go when the ops model is "restart the pod on crash" (Kubernetes default); pick Elixir when the ops model is "crashes should be invisible to users."
